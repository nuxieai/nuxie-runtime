### S3-1 — TextInput improvements

scroll velocity to the text bounds, exposes background/font properties, and
updates the authored TextInput ordering and tests. It touches focus, raw text
input, cursor behavior, and exact float/layout-visible paths. The candidate
`text_input` stream differs at the first cursor/background draw: Rust emits
the pre-change caret while C++ emits the new empty cursor path with the
background paint. This is the direct upstream prerequisite named by
#FT-TEXT, so the recommendation is **8 PORT**.

### S3-3 — static library linking

This commit moves library `require` resolution from runtime scope tables to
compile-time prelinking. It removes runtime `LibraryAsset` and the
`FileAsset.scopeLibrary{,Version}Id` fields (including `CoreUint64Type`),
uses flat prelinked module names, scopes shader/blob lookup by the calling
chunk, and revises splice/line mapping. The updated `scope_probe` is the
oracle fixture. Against the candidate C++ runner, the current Rust scripted
runner traps while registering the newly linked module
(`LUAU_ASSERT` in `lua_pushcclosurek`). The range inventory removes
`src/core/field_types/core_uint64_type.cpp`: the candidate manifest is green at
447/447, while checking that candidate-driven manifest against the d788 pin
fails on the removed old-pin row. Schema regeneration and manifest
reconciliation must therefore be part of the approved port. Because new
editor output can depend on this representation, the recommendation is
**9 PORT**.
