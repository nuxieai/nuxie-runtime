[`binary-import-completion-contract.md`](binary-import-completion-contract.md)
into a finish-line checklist. It should be updated when work changes the actual

Status meanings:

- **Verified**: authoritative current-worktree verification has passed.
- **Covered**: implementation and focused tests exist, but final completion still
  requires the verification gates to be run from the current worktree.
- **Proven in docs**: the repository has the needed scope/control artifact.
- **Open**: not yet proven by current evidence.
- **Out of scope**: deliberately excluded by the completion contract.

## Current Gaps To Close

These were the final closure checks for this goal:

- Run and record the final verification gates from the current worktree.
- If `make cpp-binary-compare` fails in a future run, classify each failure
  through the completion contract before implementing anything.
- If a future failure is in a move-later/runtime behavior family, decide whether
  it is actually needed for import parity before changing `rive-binary`.
- Keep `binary-import-surface-audit.md` updated if public helpers are moved,
  hidden, renamed, or added.
- Treat any new C++ import hook or validation hook discovered by source audits as
  import-owned only if it changes file loading, object keep/drop/null status, or
  an immediate imported relationship fact.

## Non-Blocking Runtime Work

The following may be valid future porting work, but this matrix does not require
it for `rive-schema`/`rive-binary` completion:

- Moving data-bind lifecycle helpers into a runtime crate.
- Moving converter execution/stateful interpolation into a runtime crate.
- Full dependency graph, dirt scheduling, transforms, constraints, state-machine
  execution, rendering, text layout, audio, scripting, cloning, or instancing.

Those belong to later goals.
